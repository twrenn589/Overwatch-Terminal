## What It Is

The Integrity Protocol is a **cognitive transfer methodology** — a framework for decomposing expert human judgment into layered AI systems with epistemological guardrails.

The four-layer architecture was not engineered from software best practices. It was derived by examining how a fire lieutenant actually processes a structural collapse scene — sweep the area, verify your understanding before committing, reason about what's happening behind the walls, then make the call — and encoding each cognitive function into a separate analytical layer.

The thesis monitoring is the **application**. The method is the **intellectual property**.

### The Four Layers

| Layer | Function | Fireground Analog |
|-------|----------|-------------------|
| **SWEEP** | Widest intake. No filtering, no judgment. | Arriving on scene. 360 walk-around. What do I see? |
| **CONTEXTUALIZE** | Knowledge audit BEFORE scoring. "Do I understand this well enough to evaluate it?" | Size-up. Do I understand this building type? Am I missing something? |
| **INFER** | Game theory. Who benefits? Rhetoric vs action. Circuit breakers enforce discipline. | Reading the fire. What's happening behind the walls? |
| **RECONCILE** | Final judgment. Burden of proof. Resolves contradictions. | Incident commander's decision. All information compressed into one call. |

### What Makes It Different

**Falsification-first design.** The system's circuit breakers actively prevent hallucination:
- Null Hypothesis Mandate — test the simplest explanation before inventing strategic narratives
- Assumption Count Limit — 3+ unproven assumptions = SPECULATIVE, automatically stripped from scoring
- Evidence-to-Inference Ratio — every inference must cite 2+ independent evidence points
- Skeptic Discount — speculative inferences get 75% weight reduction at Layer 4

**Cross-model oversight.** A Gemini-based Blind Auditor reads the Cognitive Trace (the full reasoning chain) without domain context. It doesn't know what XRP is. It only asks: "Does this conclusion follow from this evidence?" If Layer 4's action diverges from its own reasoning for too long, the Auditor overrides with a state-lock that only the human operator can release.

**The AI earns trust the same way a firefighter does** — through reps, not assertions. New capabilities run in shadow mode (paper trading) before getting operational authority. The evolution library tests epistemic discipline against historical scenarios with known outcomes. The corrections ledger accumulates lessons from mistakes. Trust is progressive and revocable.

## Domain-Agnostic Design

The architecture separates structure from content:

- **Schema** enforces structure (domain-agnostic) — layer ordering, gate rules, tension lifecycle, circuit breakers
- **`config/domain.json`** provides content (domain-specific) — action recommendations, thresholds, terminology
- **`thesis-context.md`** provides the thesis being monitored

To redeploy on a new domain: swap `domain.json` and `thesis-context.md`. The architecture, gates, corrections structure, and oversight mechanisms remain unchanged.

The evolution library already demonstrates this — Red Team runs a water infrastructure collapse scenario through the same pipeline that monitors a financial thesis. Same code, different domain config, different conclusions.
